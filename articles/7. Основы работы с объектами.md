# 7. Основы работы с объектами

Объекты в JavaScript — это сложные структуры данных, предназначенные для хранения коллекций различных значений в формате пар «ключ: значение». В отличие от примитивов, объекты всегда хранятся и передаются по ссылке.

---

## 1. Глубокое понимание концепции (Core Concept)

### Базовые операции со свойствами
Создать объект можно с помощью литерала `{}` (рекомендуемый способ) или конструктора `new Object()`.
* **Доступ к свойствам:** Осуществляется через точку (например, `user.name`) или через квадратные скобки (например, `user["likes birds"]`), если имя свойства содержит пробелы, спецсимволы или динамически вычисляется из переменной.
* **Проверка существования:** Обращение к отсутствующему свойству возвращает `undefined`. Для гарантированной проверки наличия ключа (даже если его значение равно `undefined`) используется оператор `in`: <code>"age" in user</code>.
* **Перебор свойств:** Для прохода по всем ключам объекта используется цикл `for...in`.

---

### Дескрипторы свойств: `Object.defineProperty`
Метод `Object.defineProperty` позволяет тонко настраивать поведение свойств. Каждое свойство объекта, помимо своего значения (`value`), имеет три скрытых атрибута (флага):
1. **`writable`** — если `true`, значение свойства можно изменить, иначе оно доступно только для чтения.
2. **`enumerable`** — если `true`, свойство перечисляется в циклах `for...in` и методах вроде `Object.keys`. Иначе циклы его игнорируют.
3. **`configurable`** — если `true`, свойство можно удалить, а его дескрипторы — изменить. Иначе этого делать нельзя.

---

### Прототипы и наследование (Prototypes)
В JavaScript наследование реализовано на прототипах. Каждый объект имеет скрытую ссылку на другой объект, называемый его прототипом.
* Эта скрытая ссылка называется `[[Prototype]]`.
* В коде получить доступ к прототипу можно через исторический геттер/сеттер `__proto__` или современные стандартные методы `Object.getPrototypeOf(obj)` и `Object.setPrototypeOf(obj, newProto)`.
* Если мы пытаемся прочитать свойство, которого нет в объекте, движок JS автоматически ищет его в прототипе этого объекта, затем в прототипе прототипа и так далее вверх по цепочке прототипов (`Prototype Chain`) до `Object.prototype`, а затем до `null`.

---

### Полезные статические методы класса `Object`
* `Object.keys(obj)` — возвращает массив ключей объекта (только собственные перечисляемые `enumerable` свойства).
* `Object.values(obj)` — возвращает массив значений свойств.
* `Object.entries(obj)` — возвращает массив пар `[ключ, значение]`.
* `Object.assign(dest, ...sources)` — копирует свойства из одного или нескольких исходных объектов в целевой.
* `Object.create(proto)` — создает новый пустой объект с указанным прототипом.

---

### Деструктурирующее присваивание
Синтаксис, позволяющий быстро извлекать свойства объекта в отдельные локальные переменные:
<code>const { firstName, lastName } = user;</code>
При деструктуризации можно переименовывать переменные: <code>const { age: userAge } = user;</code> и задавать значения по умолчанию.

---

### Копирование объектов: Поверхностное против Глубокого

Поскольку переменная хранит только ссылку на объект в памяти, простое присвоение <code>const copy = user</code> не дублирует объект, а лишь создает вторую переменную, указывающую на тот же адрес в куче.

Для полноценного дублирования данных используются два подхода:

#### А. Поверхностное копирование (Shallow Copy)
Копирует свойства только первого уровня. Если внутри объекта содержались вложенные объекты или массивы, скопируются их ссылки в памяти, и они останутся общими для оригинала и копии.
* Способы реализации: spread-оператор (<code>const clone = { ...original }</code>) или метод `Object.assign()`.

#### Б. Глубокое копирование (Deep Copy)
Рекурсивно копирует все уровни вложенности, создавая абсолютно независимые объекты, не делящие между собой память.
* Способы реализации: нативный метод `structuredClone(original)` (современный стандарт), сериализация через JSON (<code>const clone = JSON.parse(JSON.stringify(original))</code> — имеет ограничения на функции, даты и `undefined`) или метод `_.cloneDeep()` из библиотеки Lodash.

## 2. Ментальная модель

### Ссылка в оперативной памяти (Shared Reference):
```text
  [ СТЕК (Переменные-ссылки) ]             [ КУЧА (Единый объект в памяти) ]
  ----------------------------             ---------------------------------
  let userA = [Адрес #05]  ----------+
                                     |---> #05: { name: "Иван", age: 30 }
  let userB = [Адрес #05]  ----------+
  
  (Если выполнить userB.name = "Петр", изменения затронут и userA, так как объект один).
```

### Цепочка прототипов (Prototype Chain):
```text
[ Объект rabbit ] (jumps: true)
       |
       | Ссылка [[Prototype]]
       v
[ Объект animal ] (eats: true)
       |
       | Ссылка [[Prototype]]
       v
[ Объект Object.prototype ] (toString(), hasOwnProperty())
       |
       | Ссылка [[Prototype]]
       v
     null
```

## 3. Практические примеры кода

### Плохой пример: копирование объектов через присваивание и мутация дескрипторов

```javascript
const originalUser = {
  name: "Алексей",
  permissions: { isAdmin: false }
};

// Плохой подход 1: Копирование ссылки вместо объекта
const fakeCopy = originalUser;
fakeCopy.name = "Иван"; // Оригинальный originalUser тоже изменился!

// Плохой подход 2: Поверхностное копирование для вложенных структур
const shallowCopy = { ...originalUser };
shallowCopy.permissions.isAdmin = true; // Ссылка на permissions общая, оригинал сломан!

console.log(originalUser.permissions.isAdmin); // Вывод: true (утечка данных)
```

### Хороший пример: глубокое копирование и настройка неизменяемых свойств

```javascript
const originalUser = {
  name: "Алексей",
  permissions: { isAdmin: false }
};

// 1. Полноценное глубокое копирование через structuredClone
const deepCopy = structuredClone(originalUser);
deepCopy.permissions.isAdmin = true; // Безопасно, вложенный объект скопирован полностью

console.log(originalUser.permissions.isAdmin); // Вывод: false (оригинал защищен)
console.log(deepCopy.permissions.isAdmin);     // Вывод: true

// 2. Создание свойства только для чтения с помощью дескрипторов
const databaseConfig = {};

Object.defineProperty(databaseConfig, "port", {
  value: 5432,
  writable: false,      // Запрещаем перезапись значения
  enumerable: true,     // Разрешаем отображение в циклах
  configurable: false   // Запрещаем удаление свойства
});

// Попытка перезаписать свойство
databaseConfig.port = 3306; // В "use strict" вызовет TypeError, в обычном — проигнорируется
console.log(databaseConfig.port); // Вывод: 5432

// 3. Удобный перебор и деструктуризация
const stats = { speed: 80, distance: 150 };

for (const [key, value] of Object.entries(stats)) {
  console.log(`Параметр ${key} равен ${value}`);
}
```

## 4. Вопросы для собеседования

### Вопрос 1: В чем разница между поверхностным и глубоким копированием объекта?
**Ответ:** Поверхностное копирование копирует свойства только первого уровня. Если значением свойства является вложенный объект, копируется лишь его адрес (ссылка), поэтому оригинал и копия будут делить вложенный объект в памяти. Глубокое копирование рекурсивно дублирует все уровни вложенных объектов, создавая полностью автономную и независимую копию данных.

### Вопрос 2: Как работает поиск свойства по цепочке прототипов?
**Ответ:** Когда код пытается прочитать свойство объекта, движок сначала ищет его среди собственных свойств самого объекта. Если свойство не найдено, движок обращается к его скрытому свойству `[[Prototype]]` и ищет в родительском объекте. Этот процесс продолжается вверх по цепочке прототипов вплоть до `Object.prototype`. Если свойство не найдено и там, возвращается `undefined`.

### Вопрос 3: Зачем нужны флаги дескрипторов `writable` и `configurable`?
**Ответ:** 
* `writable` контролирует возможность изменения значения свойства (если `false`, свойство становится доступным только для чтения).
* `configurable` определяет, можно ли удалить это свойство из объекта с помощью оператора `delete`, а также можно ли изменить настройки самих флагов дескриптора в будущем.

### Вопрос 4: Какие ограничения имеет глубокое копирование объектов через связку `JSON.parse(JSON.stringify(obj))`?
**Ответ:** Этот метод дешев и быстр, но имеет критические ограничения: он полностью игнорирует (удаляет из объекта) свойства со значением `undefined` и функции, превращает объекты дат `Date` в строки, а также не умеет работать со специальными структурами данных (такими как `Map`, `Set`, `RegExp`) и циклическими ссылками (вызывает ошибку). Для безопасного глубокого копирования следует использовать нативный `structuredClone()`.
