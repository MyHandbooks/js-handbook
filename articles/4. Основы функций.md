Функция в JavaScript — это обособленный блок кода (подпрограмма), разработанный для решения конкретной задачи, который можно вызывать многократно из разных мест программы. Важнейшей особенностью JavaScript является то, что функции здесь — это **объекты первого класса (First-Class Citizens)**. Это означает, что функции ведут себя как обычные объекты: их можно записывать в переменные, передавать аргументами в другие функции и возвращать из функций.

---

## 1. Глубокое понимание концепции (Core Concept)

### Три способа объявления функций

#### А. Функциональное объявление (Function Declaration)
Синтаксис: <code>function sum(a, b) { return a + b; }</code>
* **Поднятие (Hoisting):** Функции этого типа полностью поднимаются движком в начало области видимости (вместе со своим телом). Это значит, что такую функцию можно вызвать в коде **выше** строки её фактического объявления.

#### Б. Функциональное выражение (Function Expression)
Синтаксис: <code>const sum = function(a, b) { return a + b; };</code>
* **Поднятие (Hoisting):** Не поднимаются. Переменная, в которую записана анонимная функция, подчиняется стандартным правилам `let` / `const` (или `var`). Вызвать её до строки объявления в коде невозможно.

#### В. Стрелочные функции (Arrow Functions)
Синтаксис: <code>const sum = (a, b) => a + b;</code>
* **Особенности:** 
  1. Стрелочные функции **не имеют своего собственного контекста `this`** (они заимствуют его из внешнего лексического окружения в момент создания).
  2. У них нет встроенного объекта `arguments` (вместо него используют rest-параметры).
  3. Их нельзя вызывать с ключевым словом `new` (они не могут быть конструкторами).

---

### Параметры против Аргументов
* **Параметры** — это переменные, перечисленные в круглых скобках при объявлении функции. Это то, что функция ожидает получить.
* **Аргументы** — это конкретные значения, которые передаются в функцию в момент её вызова.

#### Параметры по умолчанию
Если при вызове функции аргумент не передан, соответствующий параметр принимает значение `undefined`. В современном JS можно задать значение по умолчанию прямо при объявлении: <code>function greet(name = "Гость") {}</code>.

#### Остаточные параметры (Rest Parameters <code>...</code>)
Если функция принимает неограниченное число аргументов, мы можем использовать синтаксис <code>...name</code> для последнего параметра. Он соберет все оставшиеся аргументы в полноценный массив. Rest-параметр всегда должен быть последним в списке.

#### Оператор расширения (Spread <code>...</code>) в вызовах
Позволяет развернуть массив в список отдельных аргументов при вызове функции.

---

### Передача значений: по значению или по ссылке?

При передаче аргументов в функцию JavaScript ведет себя по-разному для примитивов и объектов:

* **Примитивы** (числа, строки, логические значения) передаются **по значению** (копируются). Функция получает изолированный дубликат значения в своей локальной области видимости. Изменение этого значения внутри функции никак не влияет на оригинальную переменную снаружи.
* **Объекты** (включая массивы) передаются **по ссылке** (копируется ссылка). Функция получает адрес этого объекта в куче. Если внутри функции изменить свойство объекта, **оригинальный объект за пределами функции также изменится**.

## 2. Ментальная модель

### Схема поднятия (Hoisting) функций:
```text
Область видимости при компиляции:
|---------------------------------------------------------|
| // Движок JS поднимает Function Declaration целиком     |
| function declarationFunc() { ... }                      |
|                                                         |
| declarationFunc(); // Работает корректно!               |
| expressionFunc();  // Сбой! ReferenceError (TDZ)        |
|                                                         |
| const expressionFunc = function() { ... };              |
|---------------------------------------------------------|
```

### Схема передачи аргументов в память функции:
```text
  СТЕК (Внешний контекст)                      СТЕК (Локальный контекст функции)
  -----------------------                      ---------------------------------
  let score = 100; (Примитив)   ===========>   let localScore = 100; (Копия значения)
                                               (Изменения localScore изолированы)

  let user = [Адрес #09]        ===========>   let localUser = [Адрес #09] (Копия ссылки)
                                               (Оба указывают на один объект в куче.
                                                Изменения localUser.name меняют оригинал!)
```

## 3. Практические примеры кода

### Плохой пример: побочные эффекты мутации и неверный порядок вызовов

Вызов функции выражения до ее создания приводит к падению программы, а прямая мутация переданных объектов разрушает исходные данные во внешней программе.

```javascript
// Попытка вызвать функциональное выражение до объявления
// console.log(multiply(2, 3)); // Uncaught ReferenceError: Cannot access 'multiply' before initialization

const multiply = function(a, b) {
  return a * b;
};

const myCart = { items: ["Ноутбук"], total: 1200 };

function addItemToCart(cart, newItem, price) {
  // Плохой подход: прямая мутация переданного по ссылке объекта
  cart.items.push(newItem);
  cart.total += price;
}

addItemToCart(myCart, "Мышь", 50);
// Исходный объект myCart изменился. В больших проектах это приводит к труднодоступным багам.
console.log(myCart); // Вывод: { items: ["Ноутбук", "Мышь"], total: 1250 }
```

### Хороший пример: иммутабельное изменение объектов и использование Rest-параметра

```javascript
// 1. Function Declaration можно вызывать в любой строке кода
console.log(sum(5, 5)); // Работает корректно, выведет 10

function sum(a, b) {
  return a + b;
}

// 2. Безопасное изменение объекта без мутации оригинала (создание копии)
const initialCart = { items: ["Ноутбук"], total: 1200 };

function addNewItemToCart(cart, newItem, price) {
  // Возвращаем новый объект, создавая поверхностную копию
  return {
    ...cart,
    items: [...cart.items, newItem], // Копируем массив и добавляем новый элемент
    total: cart.total + price
  };
}

const updatedCart = addNewItemToCart(initialCart, "Мышь", 50);

// Исходный объект остался нетронутым (чистая функция без побочных эффектов)
console.log(initialCart); // { items: ["Ноутбук"], total: 1200 }
console.log(updatedCart); // { items: ["Ноутбук", "Мышь"], total: 1250 }

// 3. Использование rest-параметра для сбора неограниченного числа аргументов
const calculateTotal = (tax, ...prices) => {
  const sum = prices.reduce((acc, curr) => acc + curr, 0);
  return sum + sum * tax;
};

console.log(calculateTotal(0.2, 100, 200, 300)); // Выведет 720 (налог 20% от 600)
```

## 4. Вопросы для собеседования

### Вопрос 1: В чем разница между Function Declaration и Function Expression?
**Ответ:** Основное различие заключается в процессе поднятия (hoisting):
* `Function Declaration` создается и инициализируется движком во время фазы компиляции кода. Её можно вызывать в любой строчке кода внутри её области видимости, даже выше строки с объявлением.
* `Function Expression` создается только тогда, когда поток выполнения доходит до конкретной строки с объявлением. Вызвать её раньше этой строки невозможно, так как переменная находится в TDZ или равна `undefined`.

### Вопрос 2: Как ведут себя стрелочные функции по сравнению с обычными? Назовите три ключевых отличия.
**Ответ:** Стрелочные функции имеют три важных отличия:
1. **Контекст:** У них нет собственного контекста `this`. Они заимствуют `this` из внешнего лексического окружения, в котором были созданы.
2. **Аргументы:** У них отсутствует встроенный объект `arguments` (вместо него применяются rest-параметры `...args`).
3. **Конструктор:** Их нельзя использовать в качестве конструкторов — вызов со словом `new` приведет к ошибке, у них также нет свойства `prototype`.

### Вопрос 3: Как передаются аргументы в функцию: по значению или по ссылке?
**Ответ:** Аргументы передаются по значению. Однако для сложных типов (объектов и массивов) передаваемым значением является **копия ссылки** на область памяти в куче. Из-за этого при изменении свойств объекта внутри функции оригинальный объект также меняется. Если же полностью перезаписать ссылку внутри функции (например, выполнить <code>obj = {}</code>), связь с оригинальным объектом прервется, и внешняя переменная останется прежней.

### Вопрос 4: Чем отличается оператор Rest `...` от оператора Spread `...` при работе с функциями?
**Ответ:** Они используют одинаковый синтаксис, но решают противоположные задачи:
* **Rest (остаточные параметры)** используется при *объявлении* функции. Он собирает хаотичное множество переданных аргументов в один упорядоченный массив: <code>function sum(...args) {}</code>.
* **Spread (расширение)** используется при *вызове* функции. Он, наоборот, разворачивает один массив на множество отдельных аргументов: <code>sum(...myArray)</code>.
