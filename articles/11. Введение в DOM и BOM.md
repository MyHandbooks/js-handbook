Браузерная среда предоставляет JavaScript доступ к двум ключевым объектным моделям для взаимодействия со страницей и самим браузером: **DOM (Document Object Model)** и **BOM (Browser Object Model)**. Главным связующим звеном и глобальным контекстом исполнения здесь является объект `window`.

---

## 1. Глубокое понимание концепции (Core Concept)

### 1. Глобальный объект `window`
В браузере `window` выполняет двойную роль:
1. Это глобальный объект для JavaScript (все глобальные переменные и функции, объявленные через `var`, автоматически становятся его свойствами).
2. Это представление самого окна браузера, через которое координируется работа всех остальных моделей.

---

### 2. BOM (Browser Object Model)
BOM — это объектная модель браузера. Она не имеет отношения к содержимому веб-страницы (HTML-коду), а отвечает за взаимодействие с окружением, в котором эта страница запущена.

Основные компоненты BOM:
* **`navigator`** — предоставляет информацию о самом браузере и операционной системе:
  * `navigator.userAgent` — строка идентификации браузера.
  * `navigator.language` — язык интерфейса.
  * `navigator.onLine` — булево значение, показывающее, подключено ли устройство к сети.
  * Также содержит API для работы с геолокацией (`navigator.geolocation`) и буфером обмена (`navigator.clipboard`).
* **`location`** — управляет текущим URL-адресом страницы. Позволяет получить протокол, хост, порт, параметры запроса и хэш. Присвоение значения свойству <code>location.href = "url"</code> перенаправляет пользователя на другую страницу.
* **`history`** — дает доступ к истории посещений вкладки. Методы `history.back()` и `history.forward()` имитируют кнопки "Назад" и "Вперед". В современных SPA (Single Page Applications) методы `history.pushState()` и `history.replaceState()` позволяют менять URL в адресной строке без перезагрузки всей страницы.
* **`screen`** — содержит физические параметры экрана пользователя (ширину, высоту, доступную область без учета панели задач и глубину цвета).
* **Диалоговые окна:** Методы `alert()`, `confirm()` (возвращает `true` или `false` в зависимости от выбора пользователя) и `prompt()` (возвращает введенную строку или `null`). Эти методы блокируют поток выполнения JavaScript и рендеринг страницы до тех пор, пока пользователь не закроет окно.

---

### 3. DOM (Document Object Model)
DOM — это программное представление HTML-документа в виде древовидной структуры объектов. Браузер парсит HTML-код и превращает каждый тег, текст внутри него и даже комментарии в соответствующие объекты — **DOM-узлы**.

В JS главной точкой входа в DOM является глобальный объект `document` (который сам по себе является свойством `window.document`).

#### Типы узлов в DOM-дереве:
1. **Элементы (Element nodes):** Теги HTML (например, `<body>`, `<div>`, `<p>`).
2. **Текстовые узлы (Text nodes):** Текст внутри элементов. Они всегда являются листьями дерева (у них не может быть дочерних узлов). Пробелы и переносы строк в HTML-разметке также превращаются в текстовые узлы.
3. **Комментарии (Comment nodes):** Комментарии из HTML-кода тоже парсятся в DOM, так как разработчик может динамически с ними взаимодействовать.

#### Навигация по DOM-дереву:
Для перемещения между соседними узлами используются свойства навигации. Существует два набора свойств:
* **Для работы со всеми узлами (включая текстовые и комментарии):** `parentNode`, `childNodes`, `firstChild`, `lastChild`, `previousSibling`, `nextSibling`.
* **Только для работы с элементами (тегами):** `parentElement`, `children`, `firstElementChild`, `lastElementChild`, `previousElementSibling`, `nextElementSibling`. В практической веб-разработке этот набор используется чаще всего.

---

### 4. Поиск элементов в DOM

Существует два поколения методов поиска элементов:

#### Старые (Legacy) методы:
* `document.getElementById('id')` — возвращает конкретный элемент.
* `document.getElementsByClassName('class')` — возвращает *живую* коллекцию `HTMLCollection`.
* `document.getElementsByTagName('tag')` — возвращает *живую* коллекцию `HTMLCollection`.

*Проблема "живых" коллекций:* Они автоматически обновляются при изменении DOM. Если вы нашли все элементы с классом `item`, а затем удалили один из них из DOM, коллекция моментально уменьшится в размере. Это часто приводило к багам в циклах.

#### Современные методы (на основе CSS-селекторов):
* `document.querySelector('selector')` — возвращает первый найденный элемент, соответствующий CSS-селектору, либо `null`.
* `document.querySelectorAll('selector')` — возвращает *статическую* (не обновляемую автоматически) коллекцию `NodeList` со всеми найденными элементами. Она поддерживает встроенный метод `.forEach()`.

---

### 5. Изменение элементов (DOM Manipulation)

После того как элемент найден, его можно изменить:
* **Текст:** Свойство `textContent` позволяет безопасно изменить текстовое содержимое узла. Любые переданные теги будут выведены как обычный текст (защита от XSS-атак).
* **Классы:** Свойство `classList` предоставляет методы:
  * `classList.add('className')` — добавляет класс.
  * `classList.remove('className')` — удаляет класс.
  * `classList.toggle('className')` — переключает класс (удаляет, если есть; добавляет, если нет).
* **Атрибуты:** Методы `setAttribute(name, value)`, `getAttribute(name)` и `removeAttribute(name)`.
* **Добавление элементов:** Метод `append(el)` добавляет новый элемент в конец родительского, а `prepend(el)` — в начало. Метод `remove()` полностью удаляет элемент из DOM-дерева.

## 2. Ментальная модель

Взаимосвязь моделей в браузерном окружении:

```text
                        Глобальный объект window (BOM)
                                     |
     +-------------------------------+-------------------------------+
     |                               |                               |
Объект document (DOM)         Объекты окружения (BOM)       Методы JS (Core)
     |                               |                               |
[ HTML-документ ]              - navigator                   - setTimeout()
├── <head>                     - location                    - alert()
└── <body>                     - history                     - Object
     ├── <header>              - screen                      - Array
     └── <main>
```

## 3. Практические примеры кода

### Плохой пример: использование устаревших методов поиска и небезопасное внедрение разметки

```javascript
// Попытка собрать элементы по классу старым методом
const listItems = document.getElementsByClassName("list-item");

// Небезопасное добавление содержимого через innerHTML на основе пользовательского ввода
const userInput = "<img src='x' onerror='alert(\"XSS-атака!\")'>";
const description = document.getElementById("description");

// innerHTML выполнит вредоносный скрипт, если данные не были экранированы
if (description) {
  description.innerHTML = userInput; 
}

// Попытка сделать редирект небезопасным прямым изменением истории
window.history.go(-1); // Работает грубо, пользователь может уйти со страницы неожиданно
```

### Хороший пример: современный поиск, безопасная работа с классами, текстом и BOM

```javascript
// 1. Поиск элементов по современным стандартам
const container = document.querySelector(".card-container");
const allButtons = document.querySelectorAll(".btn-action");

// NodeList поддерживает метод forEach
allButtons.forEach((button) => {
  // Безопасное переключение CSS-класса
  button.classList.toggle("btn-active");
});

// 2. Создание и безопасное наполнение нового элемента
const feedbackElement = document.createElement("p");
feedbackElement.classList.add("feedback-message", "text-success");

// textContent гарантирует защиту от XSS-инъекций
const unsafeText = "Поздравляем! <script>alert('взлом')</script>";
feedbackElement.textContent = unsafeText; 
// Результат в DOM: <p class="feedback-message">Поздравляем! &lt;script&gt;alert('взлом')&lt;/script&gt;</p>

if (container) {
  // Добавляем созданный элемент внутрь контейнера
  container.append(feedbackElement);
}

// 3. Работа с BOM для анализа окружения и безопасного перехода
if (navigator.onLine) {
  console.log("Пользователь в сети. Язык браузера: " + navigator.language);
} else {
  console.warn("Сеть отсутствует. Некоторые функции могут быть недоступны.");
}

// Безопасный редирект с возможностью вернуться назад (сохранением истории)
function redirectToSuccessPage() {
  location.assign("/success.html");
}

// Если нужно перенаправить БЕЗ возможности вернуться назад (заменить текущую запись в истории):
// <code>location.replace("/login.html");</code>
```

## 4. Вопросы для собеседования

### Вопрос 1: В чем принципиальная разница между DOM и BOM?
**Ответ:** DOM — это Document Object Model, объектное представление структуры и содержимого HTML-документа. Точкой входа является объект `document`. BOM — это Browser Object Model, предоставляющая API для взаимодействия со средой браузера (вкладки, история, экран, буфер обмена, ОС). Точкой входа является сам объект `window`.

### Вопрос 2: В чем разница между "живыми" (live) и "статическими" (static) коллекциями в DOM?
**Ответ:** "Живые" коллекции (возвращаемые методами класса `getElementsBy*`) автоматически реагируют на любые изменения DOM-дерева в режиме реального времени. Если элемент удаляется со страницы, он мгновенно пропадает из коллекции. "Статические" коллекции (возвращаемые `querySelectorAll`) фиксируют состояние DOM на момент вызова метода поиска. Последующие изменения структуры страницы не влияют на их содержимое.

### Вопрос 3: Почему метод `querySelector` считается более универсальным, чем `getElementById`? Существует ли разница в их производительности?
**Ответ:** `querySelector` универсален, так как принимает любой валидный CSS-селектор (классы, идентификаторы, атрибуты, вложенности). `getElementById` ищет строго по значению атрибута `id`. 

С точки зрения производительности `getElementById` работает быстрее, поскольку движок браузера оптимизирует поиск по уникальным идентификаторам. Однако в большинстве реальных сценариев разница в миллисекундах незаметна, поэтому гибкость CSS-селекторов в `querySelector` делает его стандартом де-факто.

### Вопрос 4: Чем отличаются методы `location.assign()` и `location.replace()`?
**Ответ:** Оба метода перенаправляют пользователя на новый URL. Разница заключается в истории посещений:
* `location.assign('url')` добавляет новую запись в историю переходов. Пользователь сможет вернуться на предыдущую страницу, нажав кнопку "Назад" в браузере.
* `location.replace('url')` заменяет текущую запись в истории новой. Предыдущая страница стирается из цепочки переходов, и вернуться на нее по кнопке "Назад" будет невозможно.
