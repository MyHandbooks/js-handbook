Для эффективного управления данными в JavaScript встроены специализированные структуры и глобальные объекты. Выбор правильного инструмента напрямую влияет на производительность приложения, читаемость кода и простоту поддержки.

## 1. Глубокое понимание концепции (Core Concept)

### 1. Массив (Array)
Массив — это упорядоченная коллекция данных, где каждый элемент имеет свой числовой индекс, начиная с `0`. В JavaScript массивы являются динамическими (их размер может изменяться автоматически) и могут хранить элементы любых типов данных одновременно.

#### Свойство `.length`
Свойство `.length` возвращает количество элементов в массиве. Важно понимать, что оно не просто считает элементы, а указывает на максимальный числовой индекс плюс один. Свойство `.length` можно перезаписывать: если уменьшить его вручную (например, присвоить <code>arr.length = 2</code>), массив будет необратимо урезан.

#### Мутирующие методы (изменяют исходный массив)
* `push(...items)` / `pop()` — добавление/удаление элемента с конца (быстрые операции).
* `unshift(...items)` / `shift()` — добавление/удаление элемента из начала (медленные операции, так как требуют переиндексации всего массива).
* `splice(start, deleteCount, ...items)` — универсальный метод для удаления, вставки и замены элементов на месте.
* `reverse()` — меняет порядок элементов на обратный.
* `sort(compareFn)` — сортирует элементы на месте (по умолчанию как строки).

#### Немутирующие методы (возвращают новый массив или значение)
* `slice(start, end)` — возвращает копию части массива.
* `concat(...items)` — объединяет массивы.
* `map(callback)` — трансформирует каждый элемент и возвращает новый массив той же длины.
* `filter(callback)` — возвращает новый массив, содержащий только элементы, прошедшие проверку.
* `reduce(callback, initialValue)` — последовательно сводит массив к единому значению (числу, строке, объекту или другому массиву).
* `find(callback)` / `findIndex(callback)` — поиск элемента или его индекса по условию.
* `includes(value)` / `indexOf(value)` — проверка наличия элемента.

#### Новые немутирующие методы (ES2023)
Классические методы вроде `sort()`, `reverse()` и `splice()` изменяют массив на месте, что часто приводит к нежелательным побочным эффектам в коде. В стандарте ES2023 добавлены их безопасные аналоги, которые возвращают копию:
* `toSorted(compareFn)` — немутирующая сортировка.
* `toReversed()` — немутирующее обращение порядка.
* `toSpliced(start, deleteCount, ...items)` — немутирующее изменение структуры.
* `with(index, value)` — заменяет элемент на указанной позиции и возвращает новую копию массива, эквивалентно операции <code>arr[index] = value</code> без изменения оригинала.

### 2. Map (Карта/Словарь)
`Map` — это коллекция пар "ключ-значение". На первый взгляд она похожа на обычный объект, но имеет принципиальные отличия:
* **Тип ключей:** В объектах ключами могут быть только строки или символы. В `Map` ключом может выступать абсолютно любой тип данных, включая числа, булевы значения, другие объекты или даже функции.
* **Порядок элементов:** `Map` всегда сохраняет порядок вставки элементов при итерации.
* **Производительность:** `Map` оптимизирована для частых операций добавления и удаления ключей.

---

### 3. Set (Множество)
`Set` — это коллекция уникальных значений (без ключей). Одно и то же значение не может быть добавлено в `Set` дважды.
* **Использование:** Идеально подходит для удаления дубликатов из массивов и быстрой проверки присутствия элемента через метод `has()`, который работает значительно быстрее, чем метод `includes()` у массивов.

### 4. Вспомогательные глобальные объекты

#### Math
Глобальный объект, содержащий константы и статические методы для математических вычислений:
* `Math.random()` — случайное число от `0` (включительно) до `1` (не включая `1`).
* `Math.floor(x)` — округление вниз к ближайшему целому.
* `Math.ceil(x)` — округление вверх.
* `Math.round(x)` — округление по математическим правилам.
* `Math.max(...args)` / `Math.min(...args)` — поиск максимального/минимального значения.

#### Date
Объект для работы с датой и временем. Время под капотом хранится в миллисекундах, прошедших с 1 января 1970 года (Unix Epoch). При работе с датами важно помнить о часовых поясах: методы вроде `getHours()` возвращают локальное время пользователя, а `getUTCHours()` — время по Гринвичу.

#### JSON
Формат передачи данных (JavaScript Object Notation). Глобальный объект `JSON` осуществляет трансформацию:
* `JSON.stringify(value)` — сериализует объект или массив в JSON-строку.
* `JSON.parse(str)` — десериализует JSON-строку обратно в JavaScript-объект.

## 2. Ментальная модель

Сравнение организации данных в памяти для базовых структур:

```text
Массив (Array) - Упорядоченная очередь в магазине
 [0: "Алексей"] -> [1: "Мария"] -> [2: "Иван"]
 (Доступ по номерку-индексу. Чтобы найти "Ивана", нужно пройти всю очередь с начала).

Множество (Set) - Коробка с уникальными деталями Lego
 { "Красная деталь", "Синяя деталь", "Зеленая деталь" }
 (Дубликаты отсекаются автоматически на входе. Быстрый поиск детали в коробке).

Карта (Map) - Гардероб в театре
 [ Билет (Объект) ]  =====>  [ Пальто (Значение) ]
 [ Ключ-карта (12) ] =====>  [ Номер отеля (Значение) ]
 (Связывает любой уникальный маркер с конкретной вещью/значением).
```

## 3. Практические примеры кода

### Плохой пример: ручная фильтрация дубликатов и использование объекта в качестве Map

Применение неподходящих структур усложняет алгоритм, замедляет работу кода и приводит к багам при приведении ключей к строкам.

```javascript
// Исходный массив с дубликатами
const users = ["Алексей", "Мария", "Алексей", "Иван", "Мария"];

// Плохой подход 1: ручной поиск дубликатов через вложенный цикл/indexOf
const uniqueUsers = [];
for (let i = 0; i < users.length; i++) {
  // Метод indexOf на каждой итерации сканирует массив заново, производительность O(N^2)
  if (uniqueUsers.indexOf(users[i]) === -1) { 
    uniqueUsers.push(users[i]);
  }
}

// Плохой подход 2: использование обычного объекта для связывания объектов-ключей
const metadata = {};
const userObjA = { id: 1, name: "Алексей" };
const userObjB = { id: 2, name: "Мария" };

// Попытка использовать объекты в качестве ключей
metadata[userObjA] = "Права: Администратор";
metadata[userObjB] = "Права: Модератор";

// Объекты неявно приводятся к строке "[object Object]"
// В итоге свойства перезаписывают друг друга по одному и тому же строковому ключу
console.log(metadata[userObjA]); // Вывод: "Права: Модератор" (данные утеряны)
```

### Хороший пример: использование Set, Map и немутирующих методов

```javascript
// Исходный массив с дубликатами
const rawUsers = ["Алексей", "Мария", "Алексей", "Иван", "Мария"];

// 1. Быстрое удаление дубликатов с помощью Set и spread-оператора
const cleanedUsers = [...new Set(rawUsers)];
console.log(cleanedUsers); // Вывод: ["Алексей", "Мария", "Иван"]

// 2. Использование Map для связывания объектов с метаданными
const userMap = new Map();
const userA = { id: 1, name: "Алексей" };
const userB = { id: 2, name: "Мария" };

// Записываем данные в Map, ключами выступают сами объекты-оригиналы
userMap.set(userA, { role: "Admin", lastActive: new Date() });
userMap.set(userB, { role: "Moderator", lastActive: new Date() });

// Чтение данных. Конфликта ключей нет, так как ссылки на объекты в памяти уникальны
console.log(userMap.get(userA).role); // Вывод: "Admin"

// 3. Безопасная работа с массивами (ES2023)
const rating = [120, 80, 450, 310];

// Сортируем копию рейтинга без мутации оригинального массива
const sortedRating = rating.toSorted((a, b) => b - a);

console.log(rating);       // Вывод: [120, 80, 450, 310] (оригинал сохранен)
console.log(sortedRating); // Вывод: [450, 310, 120, 80] (новые отсортированные данные)

// Безопасная точечная замена элемента по индексу
const updatedRating = rating.with(1, 999);
console.log(updatedRating); // Вывод: [120, 999, 450, 310]
```

## 4. Вопросы для собеседования

### Вопрос 1: В чем разница между методами `.map()` и `.forEach()` у массивов?
**Ответ:** Основная разница в возвращаемом значении и назначении. Метод `map()` применяется для трансформации данных: он вызывает колбэк для каждого элемента и возвращает новый массив той же длины, не изменяя оригинальный. Метод `forEach()` используется для выполнения побочных эффектов (например, запись в БД, логирование, изменение внешних переменных) и всегда возвращает `undefined`.

### Вопрос 2: Почему для создания ассоциативного словаря с частым добавлением и удалением элементов лучше использовать `Map`, а не обычный `Object`?
**Ответ:** 
1. В `Map` отсутствуют дефолтные ключи, унаследованные от прототипа (например, `constructor` или `toString`), что исключает случайную перезапись системных свойств.
2. `Map` позволяет использовать в качестве ключей любые типы данных, включая объекты.
3. Добавление и удаление элементов в `Map` оптимизировано на уровне движка JavaScript, поэтому эти операции выполняются быстрее, чем аналогичные манипуляции со свойствами объекта.
4. Узнать размер `Map` можно мгновенно через свойство `.size`, в то время как для объекта требуется вызывать метод <code>Object.keys(obj).length</code>, работающий за линейное время O(N).

### Вопрос 3: Как работает немутирующий метод `.with()` из стандарта ES2023?
**Ответ:** Метод `with(index, value)` заменяет элемент на указанной позиции аналогично прямому присвоению по индексу, но вместо мутации исходного массива он возвращает его поверхностную копию (shallow copy) с обновленным значением по указанному индексу. Это позволяет следовать концепции иммутабельности данных.

### Вопрос 4: Как быстро и производительно удалить все дубликаты из массива?
**Ответ:** Самый быстрый и современный способ — передать массив в конструктор `Set`, который автоматически отсечет все повторяющиеся элементы, а затем преобразовать его обратно в массив с помощью spread-оператора или метода `Array.from()`:
<code>const unique = [...new Set(array)]</code>.
